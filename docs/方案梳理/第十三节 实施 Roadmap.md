# 第十三节：实施 Roadmap 时间轴 (对应文档第13章)

**概览**：在向企业里推行这么庞大的智能化改革的时候。第一周第一月先做什么？是先把炫酷的 Agent 写出来给老板看自动回邮件吗？错。

## 这里的实施节奏非常“冷酷务实”：

### 第一个月核心军规：先建笼子再进猛虎 (阻塞式首要行动项)
第一个月不急着上线酷炫的人工智能对话。第一月必须且强行要求优先完工的是什么？
**全是安全机制。**
比如搞定 Token 预算护栏、死锁检测仪、强制错误路由归类等“保命功能”。这些底座没建完前，任何炫酷的业务功能都是空中楼阁，一票否决不准上。

### 并行的业务观察项 (非核心阻塞)
业务指标（如自动报销率到底能不能达到85%？），被当作并行观察线。也就是我们可以用真实流量慢慢看，如果初期它比较蠢，报销通过率只有40%，系统没出大乱子（没赔钱乱扣钱），就继续开着，并不阻碍全台大局子的架构竣工。

### 最关键的运维人员：Harness Engineer
队伍中有一个独立抽离出来的狠角：Harness Engineer。他**绝不**跟做大模型的去掺和怎么把大模型变得多聪明怎么写精巧提示词。他的唯一工作就是死死盯着这些做大模型的人，并把各种防呆边界、拦网给拉好，专门负责在他们的大模型出错时果断一刀切断，无情执法。

## 总结
这套推进路网，明确把“炫技”（AI 提效）与“保命”（系统防御底盘）分开核算，防守团队权重极高，确保绝不出大规模的生产安全与合规灾害。
