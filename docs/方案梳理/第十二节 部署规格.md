# 第十二节：部署规格与技术栈 (对应文档第12章)

**概览**：整套系统并不是非要买一百台服务器才能转得起来，架构本身像变压器一样具备非常高的伸缩弹性。

## 部署的分为三等模式：

1.  **极简小作坊版（规格A）：适用于老板想看POC效果，5个人小团队。**
    *   能砍的周边防爆设备全砍了。不需要庞大的ClickHouse集群来支持万年不挂的审计功能，普通关系数据库 PG 顶替；那些重型杀伤性的极度防爆沙箱虚拟机不运行了，大家都彼此相信是“内信任环境”。

2.  **企业稳步上线版（规格B）：适用于内部推广，需要 8-9 人。**
    *   该有的都有，把核心拦截 Harness 的功能上全，普通级别的隔绝容器跑起来。它能应付日常没有故意破坏环境下的高压任务。

3.  **金融纯血多活版（规格C）：涉及核心票款审批、高并发，需十多名资深架构师撑场。**
    *   万物皆为强隔离，核弹级安全沙箱全速开启，各种Redis集群、Patroni热备机制上线防止连网线被挖断。它必须像银行后台一样，任何时刻不能出死锁不能漏一条审计残渣。

## 总结
这套架构在技术选型上非常明确：抛弃花坛上五彩斑斓但不扛造的各类“新兴潮玩AI小工具库”。极其硬汉式地采用了类似 Kong 网关、Kafka 列队、Patroni 集群这种传统但经过亿万次金融生死考验的工业级传家宝来作为地基承接上方大模型的骚套路。
