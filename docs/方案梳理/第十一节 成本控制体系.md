# 第十一节：成本控制体系 (对应文档第11章)

**概览**：做大模型落地往往都是从一个很好的创意开始，以一份老板震怒的天价对账单收场。成本控制体系是把这只“吞金兽”关回笼子的财务算盘。

## 1. 成本花在了哪里？
只要发给大模型的字，以及它吐出来的片段哪怕没有被我们采用，它全部都要按字符（Token）算钱。而在企业内做多层检索对比、或者做死循环发疯的时候，Token 消耗是指数级别的无底洞。同时系统长期日积月累保存的庞大的多模态文档记录和日志，让最昂贵的热数据库（内存及高端硬盘）空间告急。

## 2. 怎么“扣门”地控点？

### 2.1 三六九等的算力分配（Token分层计费）
*   并非凡事都要动用深邃的GPT4（也就是高配大模型，被分为 Tier C），贵得可怕。
*   能用便宜的轻量版模型比如一些专管提取短分类信息的（Tier B），就绝不动用主力大管家。
*   凡是能在第一轮规则引擎（比如看看权限符不符合，白名单在不在）（Tier A）就拦截甚至识别的内容，不用耗费任何 Token（等于不要钱）立即返回。
*   当系统观察到一个对话主任务，把钱烧到 70% 的危险边缘了，系统会强行把后续的动作全部降级给只配廉价的小模型草草去收尾办，办不了就强制熔断报警（见第二章Harness成本阀门体系）。

### 2.2 冷热存储轮换（极度压榨存储单价）
所有的记录都必须做终极审计对账保留的，可长期霸占着ClickHouse（顶级性能数据库）也是巨大的浪费。
系统设置了到期衰变期。比如超过90天无人问津的前期古老审计与思维链路，它会被像移入档案馆一样自动转储入巨型且巨便宜的云端 OSS 慢速存储桶里面压缩打包冷藏。要查时再捞出来即可，日常几乎无负担。

## 总结
将花大模型的计算银子像采购钢筋水泥一样按照工程等级区分使用（最重磅承重梁才上Tier C 主角），极佳地平衡了智能的逼格与老板的预算。
